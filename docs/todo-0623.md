## 🔴 Bug

- [x] **Thread 消息顺序错乱**
      助理回复的文本出现在工具调用之前。
      根因：`streamChat` 先发 `assistant_delta`，`tool_call` / `tool_result` 在 `streamChat` 返回后才发。
      方案：streamChat 返回前 emit tool_call + Agent 标志防 dispatchAndDetect 双发。测试全通过。

- [x] **历史会话不完整（测试中发现）**
      `/api/history` 只返回用户消息，缺少助理消息、工具调用链、思维链。
      根因：(1) `handleHistory` 的 `Msg` 结构体过简，丢弃了 `ToolCalls`、`ToolCallID`、`ReasoningContent`；(2) 前端 `loadHistory` 跳过无 content 的 assistant 消息；(3) `streamChat` 未将 reasoning 存入 session。
      方案：三层修复——存储层加 ReasoningContent、API 层扩展 Msg 结构体、前端 loadHistory 完整渲染。

- [x] **thinking + 工具折叠组未生效**
      `turn-group` 容器未正确渲染且不能折叠收纳。
      根因：(1) `toggleTurnGroup` 用 `el.parentElement`（thread）而非 `el`（turn-group）操作 class，CSS `.turn-group.collapsed` 永远不匹配；(2) `scrollToBottom` 函数缺失；(3) `addMsg('notice')` 重置了 `currentTurnGroup`；(4) `addToolRow` 没有主动创建 turn-group。
      方案：修正 `toggleTurnGroup` 作用域、补充 `scrollToBottom`、notice 不打断分组、`addToolRow` 改用 `ensureTurnGroup()`。

- [x] **Markdown 渲染未生效**
      `appendAssistant` 和 `addMsg('assistant')` 中的 `renderMarkdown` 调用未显示效果。
      根因：`renderMarkdown` 产出 `<strong>`、`<code>`、`<pre>`、`<h2>` 等 HTML 元素，但 CSS 中没有任何对应样式，深色主题下不可见。
      方案：补齐所有 Markdown 元素的 CSS 样式（strong/code/pre/h2-h4/ul/li/hr/p）。

- [ ] **Plan 模式 Web 审批流程不完整**
      - 生成方案后确认框缺少详细步骤展示
      - `GeneratePlan` / `ReviewPlan` 为同步 LLM 调用，耗时较长无进度反馈
      - 用户易误以为卡死而切换模式
      方案：Plan 生成期间显示「正在生成方案…」动画；确认框展示完整步骤列表。

- [ ] **mode 切换偶发失效**
      快速多次点击 Status Bar 的 mode 标签可能导致状态不同步。
      方案：`toggleMode` 加防抖或锁定，防止并发请求。

## 🟡 功能缺口

- [ ] **Status Bar 展示 plan_mode**
      `state` 事件已含 `plan_mode` 字段，但前端未展示。
      方案：在 Status Bar mode 标签旁加「🔒 只读」标记。

- [ ] **`/api/tools` 端点**
      Agent 端 `ListTools()` 已实现，但 HTTP 端点和前端展示未连通。
      方案：注册路由 + 前端展示可用工具列表。

- [ ] **工具执行超时机制**
      `executeToolCall` 没有 context timeout，MCP 工具卡住会阻塞整个 Agent。
      方案：`context.WithTimeout` 包裹工具执行，默认 60s。

- [ ] **风暴检测可靠性**
      `isErrorResult` 靠关键词匹配错误，易漏判或误判。
      方案：工具返回结构化结果 `{ok, error, data}`。

- [ ] **`all_tools.go` 拆分**
      15 个工具实现在一个文件（~22000 字节），建议按职责拆分。
      方案：拆为 file/edit/search/shell/git/web/todo 等独立文件。

- [ ] **`main.go` 命令注册抽离**
      大量命令 Handler 在 main.go（~20000 字节）。
      方案：命令 Handler 定义抽到 `commands.go`。
