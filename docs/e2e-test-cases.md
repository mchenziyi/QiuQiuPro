# QiuQiuPro E2E 测试用例

## 测试环境
- Agent 模式：ask（默认）/ plan
- 模型：DeepSeek V4
- 操作系统：Windows (本机) / macOS (用户)

---

## TC01 — Ask 模式·简单问答

**前置条件**：agent 启动，默认 ask 模式

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入 `你好，介绍一下自己` | 模型自我介绍，说明是球球（QiuQiuPro）Coding Agent |
| 2 | 检查输出 | 回答里包含能力和工具介绍 |
| 3 | 检查 token | 有 `📊 本轮 token` 行 |

**通过条件**：模型正常回答，无报错

---

## TC02 — Ask 模式·读文件

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入 `看看 agent/agent.go 的前20行` | 模型调用 `read_file` 或 `glob`+`read_file` |
| 2 | 检查输出 | 返回文件内容，显示正确行数 |
| 3 | 检查工具调用 | 日志中有 `🔧 read_file` 或 `🔧 glob` |

**通过条件**：工具调用成功，文件内容正确返回

---

## TC03 — Ask 模式·代码搜索

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入 `搜一下 agent 包里哪里用到了 SetPlanMode` | 模型调用 `grep("SetPlanMode")` |
| 2 | 检查结果 | 返回所有引用位置（agent.go、skill.go、main.go 等） |

**通过条件**：grep 搜索结果准确，模型正确解释

---

## TC04 — Ask 模式·写文件

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入 `写一个 test.txt 文件，内容为 hello` | 模型调用 `write_file({"path": "test.txt", "content": "hello"})` |
| 2 | 确认文件存在 | 磁盘上生成 test.txt，内容为 hello |
| 3 | 清理 | `del test.txt` |

**通过条件**：文件被创建且内容正确

---

## TC05 — 命令系统

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入 `/help` | 显示帮助信息，列出所有可用命令 |
| 2 | 输入 `/mode` | 显示当前模式 |
| 3 | 输入 `/mode plan` | 切换到 plan 模式，提示符变为 `[PLAN]` |
| 4 | 输入 `/mode ask` | 切换回 ask 模式，提示符变回 `[ASK]` |
| 5 | 输入 `/readonly` | 显示当前只读状态 |
| 6 | 输入 `/readonly on` | 开启只读，提示符变为 `[🔒ASK]` |
| 7 | 输入 `/readonly off` | 关闭只读 |
| 8 | 输入 `/memory` | 显示"暂无长期记忆"或记忆列表 |
| 9 | 输入 `/unknown` | 显示"未知命令"提示 |
| 10 | 输入 `exit` | Agent 退出，显示"再见" |

**通过条件**：所有命令正常响应

## TC06 — 只读模式·拒绝写操作

**前置条件**：`/readonly on`

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入 `写一个 test.txt` | 模型调用 write_file → 权限门返回 `GateDeny` |
| 2 | 检查输出 | 显示 `🚫 已拒绝执行` 或 `blocked` |
| 3 | 验证 | 文件未被创建 |

**通过条件**：写工具被明确拒绝，文件不存在

---

## TC07 — Plan 模式·只读调研

**前置条件**：`/mode plan`

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入 `看看 main.go 的架构` | 模型识别 plan 模式，只用只读工具（read_file/glob/grep） |
| 2 | 检查输出 | 模型分析 main.go 结构，不尝试写文件 |
| 3 | 检查提示符 | 显示 `[PLAN]` |

**通过条件**：模型在 plan 模式下正确使用只读工具

---

## TC08 — Plan 模式·方案审批流程

**前置条件**：`/mode plan`

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入一个带修改需求的任务 | 模型调研后输出方案文本 |
| 2 | 检查 `批准执行？[Y/n]` 提示 | 显示审批确认 |
| 3 | 输入 `n` | 方案被取消，保持 plan 模式 |
| 4 | 重新输入，输入 `Y` | 方案被批准，plan 模式关闭，开始执行 |

**注意**：此场景依赖交互式 stdin，管道测试受限，建议人工测试

**通过条件**：审批流程完整

## TC10 — 错误处理

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 输入空行 | 忽略，不消耗 token |
| 2 | 输入 `///` | 非命令，作为用户消息发送给模型 |
| 3 | 输入超长文本（>10000 字） | 模型正常处理或压缩 |

**通过条件**：系统不崩溃，正常处理边界输入
