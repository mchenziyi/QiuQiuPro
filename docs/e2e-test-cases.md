## 一、Agent 模式（3 用例）

### TC-AGENT-01：Ask 模式—直接问答
| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | 输入 `你好` | 模型回复问候 |
| 2 | 输入 `今天天气怎么样（不需要联网）` | 模型以自身知识回答，不调用工具 |
| 3 | 检查 token 行 | 显示 `📊 本轮 token`，无工具调用日志 |
| 4 | 连续对话 3 轮 | 上下文保持连贯，每轮有 token 报告 |

### TC-AGENT-02：Plan 模式—只读调研
| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | `/mode plan` | 提示符变为 `[PLAN]` |
| 2 | `帮我看看 main.go 的代码结构` | 模型使用 read_file / grep 等只读工具调研 |
| 3 | 模型不调 write 类工具 | 日志中没有 write_file / edit_file / bash 等 |
| 4 | 模型输出分析结论 | 回答中包含对 main.go 架构的分析 |

### TC-AGENT-03：Plan 模式—审批→执行链
| # | 操作 | 预期结果 |
|---|------|---------|
| 1 | `/mode plan` | 进入 plan 模式 |
| 2 | 输入一个修改需求（如 `在 main.go 加个版本号常量`） | 模型用只读工具调研，输出方案 |
| 3 | 看到 `批准执行？[Y/n]` 提示 | 出现审批确认 |
| 4 | 输入 `n` | 取消执行，保持 plan 模式 |
| 5 | 输入 `Y` | plan 模式关闭，开始执行修改 |
| 6 | 检查文件变更 | main.go 已修改 |
