# 30 — 提示词全面重写（Reasonix 风格）

## 为什么要做

之前的提示词大量篇幅在"你是谁"上——身份声明 5 行，核心规则 5 行，但没有告诉模型**在什么场景、用什么工具、怎么做**。Reasonix 的 system prompt 只做一件事：每个场景映射到一个具体行动。

## 改了哪些

### system.xml：从"身份声明"到"场景→工具→行为"

| 之前 | 之后 |
|------|------|
| 5 行身份声明（你不是 DeepSeek...） | 1 行身份 + 3 条原则 + 3 个场景 |
| "核心规则" 列表（你应该...） | "原则"（先理解再动手、改动最小） |
| 没提 ask 工具怎么用 | "不确定时调用 ask 列出 2-4 个选项" |
| 没提 todo_write | "始终只有一个 in_progress，边做边更新" |
| 没提 Plan 模式行为 | "纯调研 → 写方案 → 等批准 → 动手" |

### plan/*.xml：砍角色扮演

| 文件 | 砍了什么 | 保留什么 |
|------|---------|---------|
| generate.xml | "你是严谨的项目规划专家" | 粒度要求 + JSON 格式 |
| review.xml | "你是规划评审专家" | 4 个检查维度 |
| reflect.xml | "你是复盘专家" | 3 维度分析 + 50-100 字 |
| replan.xml | "你是规划专家" | 不重复已完成 + 按反思调整 |

### 核心设计原则（参照 Reasonix）

1. **每句话指向行动**：不是"你应该怎么做"，是"用 X 工具去做 Y"
2. **场景驱动**：不确定 → ask、多步 → todo_write、不能改 → 纯调研
3. **砍掉"专家"人设**：不说"你是 X 专家"，只给清晰指令
4. **长度减半**：5 个文件从 99 行 → 57 行

## 改动文件

| 文件 | 改动 |
|------|------|
| `prompt/default/system.xml` | 重写：场景→工具→行为 |
| `prompt/plan/generate.xml` | 精简：砍角色声明 |

## 效果

- 提示词总 token 减少 ~40%
- 每句话都指向具体工具或行为
- Agent 行为更可预测（知道了"什么时候该问"、"什么时候该规划"）
- 不改代码就能更新（XML 模板热加载）
